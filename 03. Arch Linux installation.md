# 2. Дискове
## Разделяне на дяловете
Нека да видим какво имаме като дискове и дялове с
```bash
lsblk
```

Двам пример с лаптоп Thinkpad T14s Gen1 с ssd 256 GB.  
Инсталирани са вече Windows 11 и Endeavour OS.  
Целта е да се изтрие Endeavour OS и да се инсталира Arch Linux на неговото място.

```bash
# Windows 11 - дялове които остават
    /dev/nvme0n1p1 – EFI (Windows boot)
    /dev/nvme0n1p2 – MSR
    /dev/nvme0n1p3 – BitLocker Windows C:
    /dev/nvme0n1p4 – Windows recovery
# EndeavourOS - дялове които се трият
    /dev/nvme0n1p5 – /efi (на EndeavourOS)
    /dev/nvme0n1p6 – неизвестен (също EndeavourOS)
    /dev/nvme0n1p7 – swap
    /dev/nvme0n1p8 – /
    /dev/nvme0n1p9 – /home
# Arch Linux - дялове които се създават
    / - 60 GB
    /home - 60-70 GB
    swap - 8GB
    EFI - 256-512 MB only if this in the one operating system for the device
```

## Delete partition
### from old Endeavour OS
```bash
sudo fdisk /dev/nvme0n1
p - показва дяловете

d - трие и пита кой номер
5

w - записва промените
```

## Create partitions
```bash
# Root
sudo fdisk /dev/nvme0n1
n - създава дял
5
<default first sector>
+60G

    # Partition 5 contains a vfat signature 
    # remove signature yes or no
    y
```
```bash
    Какво е „подпис“ (signature)
        Това е малък маркер в началото на дяла, 
        който казва каква файлова система е била там:
            vfat
            ext4
            ntfs
            Swap
```
```bash	
# Swap
sudo fdisk /dev/nvme0n1
n - създава дял
7
<default first sector>
+8G - за Swap
```

```bash
# Home
sudo fdisk /dev/nvme0n1
n - създава дял
6
<default first sector>
<default size> - да вземе останалото място
```
## Format partitions
```bash
# / (root)
mkfs.ext4 /dev/nvme0n1p5

# /home
mkfs.ext4 /dev/nvme0n1p6

# Swap
mkswap /dev/nvme0n1p7

# EFI
mkfs.fat -F32 /dev/nvme0n1p1
```

## Mounting
```bash
# / (root)
mount /dev/nvme0n1p5 /mnt

# /home
mkdir /mnt/home
mount /dev/nvme0n1p6 /mnt/home

# Swap
swapon /dev/nvme0n1p7

# EFI
mkdir -p /mnt/boot
mount /dev/nvme0n1p1 /mnt/boot

# за проверка
lsblk
```

# 3. Инсталиране на Arch Linux
Много е важно да са mount-нати всичките 4 точки от раздела Mount
```bash
# Инсталиране на Arch Linux
pacstrap /mnt base linux linux-firmware

# Генериране на fstab
genfstab -U /mnt >> /mnt/etc/fstab

# Провери
cat /mnt/etc/fstab

# Влизаш в новоинсталираната система върху диска.
arch-chroot /mnt  
    # Arch ISO остава само "инсталатор", а chroot те прехвърля вътре в истинския Arch.
    # Оттук нататък всички команди се изпълняват все едно системата вече е стартирана от диска.
```


# 4. Grub boot loader
## Инсталираме Grub boot loader
```bash
# Влизаме в системата chroot - [root@archiso ~#]
arch-chroot /mnt
```
Пуска се от системата chroot - [root@archiso ~#]
```bash
pacman -Sy grub efibootmgr
grub-install --target=x86_64-efi --efi-directory=/boot --bootloader-id=GRUB
grub-mkconfig -o /boot/grub/grub.cfg

efibootmgr --create --disk /dev/nvme0n1 --part 1 --label "Arch Linux" --loader "\EFI\GRUB\grubx64.efi"

efibootmgr

exit
reboot
```
# 5. Потребител - настройки
Пуска се от системата chroot - [root@archiso ~#] с
```bash
/2_user.sh
```
```bash
#!/bin/bash

# 0) Sudo
pacman -S --noconfirm sudo

# 1) Font
pacman -S --noconfirm terminus-font
#setfont ter-v32b
#echo "FONT=ter-v32b" > /etc/vconsole.conf

setfont ter-228n
echo "FONT=ter-228n" > /etc/vconsole.conf

# 2) Wi-Fi (iwd)
pacman -S --noconfirm iwd
systemctl enable iwd
systemctl start iwd

# 3) NetworkManager
pacman -S --noconfirm networkmanager
systemctl enable NetworkManager

# 4) User
useradd -m -G wheel peter

# 5) sudo for wheel group
sed -i 's/# %wheel ALL=(ALL:ALL) ALL/%wheel ALL=(ALL:ALL) ALL/' /etc/sudoers
```
Ако 5та точка не проработи, се добавя потребителя ръчно
```bash
usermod -aG wheel peter
EDITOR=nano visudo
# Търси се ред %wheel ALL=(ALL:ALL) ALL
# и се разкоментирва
```
